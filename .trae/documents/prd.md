## 1. 产品概述

招聘简历解析与智能匹配系统，面向企业HR和求职候选人，通过大模型API自动解析PDF简历提取结构化数据，结合语义向量相似度实现岗位与候选人的智能匹配，提升招聘筛选效率。
- 解决传统人工筛选简历效率低、匹配标准不统一的问题，目标将初筛效率提升3倍以上

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 候选人 | 邮箱注册 | 上传简历、查看解析结果、编辑确认个人信息 |
| HR | 邮箱注册 | 发布岗位、查看匹配候选人、设置状态标签、添加备注、导出列表 |

### 2.2 功能模块

1. **简历上传与解析页**：PDF上传、大模型结构化提取、二次确认编辑
2. **岗位管理页**：发布岗位、填写JD、编辑/关闭岗位
3. **岗位匹配详情页**：候选人列表按匹配分排序、高亮匹配点与差距项、状态标签、备注
4. **候选人筛选与导出页**：按状态和匹配分区间筛选、导出Excel

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 简历上传与解析 | PDF上传区 | 拖拽或点击上传PDF简历，支持单文件上传 |
| 简历上传与解析 | 结构化信息展示 | 展示大模型提取的基本信息、教育经历、工作经历、技能标签 |
| 简历上传与解析 | 二次确认编辑 | 候选人可逐字段编辑修改解析结果并保存 |

| 岗位匹配详情 | 候选人列表 | 按匹配分从高到低排序展示候选人卡片 |

| 候选人筛选与导出 | 导出Excel | 一键导出筛选后的候选人列表为Excel文件 |

## 3. 核心流程

**候选人上传简历流程**：候选人注册登录 → 上传PDF简历 → 系统调用大模型API提取结构化信息 → 展示解析结果 → 候选人二次确认编辑 → 保存简历信息

**HR智能匹配流程**：HR注册登录 → 发布岗位（填写JD） → 系统计算岗位与所有候选人简历的语义相似度 → 岗位详情页展示候选人按匹配分排序列表 → HR查看匹配点/差距项 → 设置候选人状态标签 → 添加内部备注 → 筛选候选人 → 导出Excel

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝 (#1E3A5F) 搭配活力橙 (#FF6B35) 作为强调色，传达专业与活力
- 辅助色：浅灰背景 (#F7F8FA)、白色卡片、绿色匹配 (#22C55E)、红色差距 (#EF4444)
- 按钮风格：圆角矩形，主按钮填充橙色，次按钮描边
- 字体：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular
- 布局风格：左侧导航栏 + 右侧内容区，卡片式布局
- 图标风格：使用 Lucide 图标库，线性风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 简历上传与解析 | PDF上传区 | 虚线边框拖拽区，上传图标，文件名展示，上传进度条 |
| 简历上传与解析 | 结构化信息展示 | 白色卡片分区展示（基本信息/教育/工作/技能），标签式技能展示 |
| 简历上传与解析 | 二次确认编辑 | 行内编辑模式，保存/取消按钮，编辑状态视觉反馈 |
| 岗位管理 | 岗位列表 | 卡片网格布局，岗位状态标签，快捷操作按钮 |
| 岗位管理 | 发布岗位表单 | 模态框表单，多行文本框输入JD，标签式技能输入 |
| 岗位匹配详情 | 候选人列表 | 纵向卡片列表，匹配分环形进度指示，排序指示 |
| 岗位匹配详情 | 匹配分析 | 绿色高亮匹配标签，红色高亮差距标签，悬浮详情面板 |
| 岗位匹配详情 | 状态标签 | 下拉选择器（初筛/面试/发Offer/淘汰），彩色状态徽章 |
| 岗位匹配详情 | 内部备注 | 可展开备注区，时间戳标注，编辑/保存切换 |
| 候选人筛选与导出 | 筛选面板 | 顶部筛选条，多选复选框，滑块式匹配分区间选择 |
| 候选人筛选与导出 | 导出Excel | 橙色导出按钮，导出进度提示，文件下载 |

### 4.3 响应式设计

- 桌面优先设计，最小宽度1024px
- 侧边导航可折叠，内容区自适应
- 卡片布局在窄屏下从多列变为单列
